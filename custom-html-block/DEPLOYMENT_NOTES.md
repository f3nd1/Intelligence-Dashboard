# How to Add or Remove Graphs

## 1. Scope

This guide applies to the UCC Intelligence Platform v1.9.11 structure.

Most graphs are controlled from:

```text
JAVASCRIPT.js
```

Criterion 4 also has several main diagrams controlled from:

```text
UCC Analytics - Criterion 4.py
HTML.html
```

> Do not add new CSS merely to add or remove a graph. Existing graph cards, layouts and visual styles should be reused.

---

# 2. Graph Architecture

The platform has three graph configurations.

| Criteria | Configuration | Location |
|---|---|---|
| Criterion 1, 2, 3, 5, 6 and 7 | Generated live graphs | `LIVE_VISUAL_EXPANSION` in `JAVASCRIPT.js` |
| Criterion 4 additional graphs | Generated live graphs | `C4_VISUAL_EXPANSION` in `JAVASCRIPT.js` |
| Criterion 4 main diagrams | Exact metric-to-stage diagrams | `C4_VISUAL_REGISTRY` in `UCC Analytics - Criterion 4.py` |

Use the generated live graph configuration for normal additions or removals.

Only use the Criterion 4 Python registry when a diagram requires exact metric IDs and exact process stages.

# 5. Graph Configuration Fields

| Field | Required | Purpose |
|---|---:|---|
| `id` | Yes | Unique identifier for the graph |
| `title` | Yes | Heading displayed above the graph |
| `type` | Yes | Visual type, such as `bar`, `donut` or `radar` |
| `description` | Recommended | Short explanation shown beside the title |
| `i` | Recommended | Controls which set of available metrics is used |
| `enabled` | Optional | Set to `false` to hide the graph |

## ID rules

Use a unique and meaningful ID.

Recommended format:

```text
criterion-section-purpose
```

Examples:

```text
c5-511-curriculum-readiness
c2-211-staff-status
c6-631-improvement-trend
```

Do not reuse an existing graph ID.

## Understanding `i`

The `i` value selects a different group of available metrics.

Example:

```javascript
"i": 0
```

uses the first metric group.

```javascript
"i": 1
```

uses the next metric group.

```javascript
"i": 2
```

uses another metric group.

Each normal graph can display up to five available metrics.

Change `i` when two graphs are displaying the same values and should show different metric groups.

---

# 6. Special Graph Titles

Some title phrases automatically use special summary data instead of the normal metric group.

## Source summary graphs

Titles containing the following phrases use source and metric availability totals:

```text
Source Availability
Evidence Readiness
Source Readiness
```

Example:

```javascript
{
  "id": "c5-511-source-readiness",
  "title": "Course Design Source Readiness",
  "type": "donut",
  "description": "Shows readable sources and source issues.",
  "i": 0
}
```

## Status and health graphs

Titles containing the following phrases use available metrics, unavailable metrics, sources and exceptions:

```text
Status Distribution
System Health
Control Health
Readiness
```

## Exception graphs

Titles containing the following phrases use exception metrics:

```text
Exception
Gap
Risk Profile
```

Be careful when naming a graph because these phrases affect which data is displayed.

# 7. Remove a Graph

## Recommended method: Disable the graph

Add:

```javascript
"enabled": false
```

Example:

```javascript
{
  "id": "c5-511-curriculum-readiness",
  "title": "Curriculum Readiness",
  "type": "radar",
  "description": "Compares curriculum and assessment readiness.",
  "i": 4,
  "enabled": false
}
```

This preserves the configuration while removing the graph from the interface.

## Permanent removal

Delete the entire graph object:

```javascript
{
  "id": "c5-511-curriculum-readiness",
  "title": "Curriculum Readiness",
  "type": "radar",
  "description": "Compares curriculum and assessment readiness.",
  "i": 4
}
```

Ensure that the previous object still ends with a valid comma where required.

Change only the `type` value.

Before:

```javascript
  "id": "c5-511-coverage",
  "title": "Course Design Control Coverage",
  "type": "bar",
  "description": "Compares the available course design controls.",
  "i": 0
}
```

After:

```javascript
{
  "id": "c5-511-coverage",
  "title": "Course Design Control Coverage",
  "type": "radar",
  "description": "Compares the available course design controls.",
  "i": 0
}
```

Do not change the graph ID merely because the visual type changes.

---

# 9. Graph Display Order

Graphs are displayed alphabetically according to their `title`.

Changing the object order inside the JavaScript array may not change the displayed order.

For a required order, use clear titles such as:

```text
1. Control Coverage
2. Status Distribution
3. Evidence Readiness
4. Exception Profile
```

Only add numbering when a fixed business sequence is necessary.

# 12. Add a Completely New Criterion 4 Main Diagram

A new main Criterion 4 diagram requires coordinated changes.

## A. Add the Python registry entry

```python
"c411-new-control": {
    "panel": "c411",
    "type": "network",
    "title": "Admissions Evidence Network",
    "stages": [
        {
            "label": "Counselling",
            "metric": "c411-counselling",
            "source": "Pre Course Counselling Declaration"
        },
        {
            "label": "Application",
            "metric": "c411-complete",
            "source": "Student Applicant"
        }
    ]
}
```

## B. Add the HTML diagram container

In `HTML.html`:

```html
<div
  class="ucc-c4-d3-visual"
  data-c4-card-panel="diagram"
  data-c4-visual="c411-new-control"
>
  <div class="ucc-c4-visual-loading">Preparing live diagram…</div>
</div>
```

## C. Add the matching table container

```html
<div class="table-wrap hidden" data-c4-card-panel="table">
  <table class="ucc-c4-visual-table">
    <thead>
      <tr>
        <th>Stage / control</th>
        <th>Current count</th>
        <th>Status</th>
        <th>Source</th>
        <th>Matching records</th>
      </tr>
    </thead>
    <tbody data-c4-visual-table="c411-new-control"></tbody>
  </table>
</div>
```

The following values must match exactly:

```text
data-c4-visual="c411-new-control"
data-c4-visual-table="c411-new-control"
Python registry key: "c411-new-control"
```

only update:

- a new source or field mapping is required;
- Criterion 4 needs an exact stage-to-metric diagram;
- the API response structure must change.

# 14. Validation Checklist

After adding, removing or changing a graph:

- [ ] The graph ID is unique.
- [ ] The criterion key is correct.
- [ ] The section key matches the actual tab.
- [ ] The graph type is supported.
- [ ] The JavaScript object has valid commas and brackets.
- [ ] No unnecessary CSS was added.
- [ ] The graph appears when the section is opened.
- [ ] Diagram view loads without clicking another menu first.
- [ ] Table view displays the same live values.
- [ ] Filters refresh the graph.
- [ ] Source Mapping Report shows no new rendering issue.
- [ ] Existing Criteria 1 to 7 still open normally.
- [ ] Analytics, Explore and Ask UCC still work.
- [ ] Browser cache was cleared or a hard refresh was performed.

# 15. Deployment

After updating the files:

1. Replace the matched platform files.
2. Clear the Frappe cache.
3. Reload the page using a hard refresh.
4. Open the affected criterion.
5. Test the graph in both Diagram and Table views.
6. Test at least one different criterion to confirm no regression.

Do not deploy only a partial CSS or JavaScript patch when the release also contains coordinated HTML or Python changes.
