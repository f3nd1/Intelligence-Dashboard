# Custom HTML Block Deployment Notes - v1.9.6

This is a frontend-only hotfix. Replace all three Custom HTML Block fields together:

1. HTML: `HTML.html`
2. CSS: `CSS.css`
3. JavaScript: `JAVASCRIPT.js`

Do not replace any Server Script if v1.9.5 Server Scripts are already installed.

After saving:

1. Clear the ERPNext/Frappe cache.
2. Hard refresh Sophia.
3. Confirm the header displays `v1.9.6`.
4. Open View tools and confirm the menu appears.
5. Click Source mapping report and confirm the styled report opens.
6. If a deployment mismatch warning appears, one of the three frontend fields is still stale. Replace all three fields again.

The `survey.min.css` 404 and `custom_calendar_override9.js` error are separate site-level issues and are not generated by this Custom HTML Block.
