# Troubleshooting

## Ask UCC says method not found

Confirm the API methods are exactly:

```text
ucc_ask_student_journey
ucc_ask_recruitment_agent
ucc_ask_quality_action
```

The visible Server Script name can contain spaces; the API method cannot.

## Dashboard is blank

Check:

1. browser console;
2. Diagnostics Log;
3. D3 network request;
4. read permission on Academic Year, Student Group, Course and Program;
5. HTML/JavaScript build mismatch warning.

## A source says unavailable

Unavailable may mean:

- DocType does not exist;
- user lacks permission;
- a requested field does not exist;
- translated and technical names differ;
- the live site schema differs from the repository snapshot.

Do not change unavailable to zero.

## Charts appear but tables are wrong

Use the same filters, open the underlying records and compare the calculation documented in `DATA_SOURCE_MAP.md`.

## Ask UCC record search is empty

Student Journey currently depends on the Student Roll report path in the source frontend. Confirm the report exists and the user can run it.

Recruitment Agent and Quality Action use direct permission-aware list calls.

## OpenAI mode does not work

Guided questions do not need OpenAI.

For free-form mode:

- verify the key was entered for the current browser session;
- verify the Server Script allows the request format;
- check network and provider errors;
- do not hard-code a permanent key in JavaScript.

## CSS affects other Frappe content

A selector may be too broad. New styles must be prefixed with `.ucc-platform`, `.ucc-c5-v41` or `.aja-app`.

## Criterion selector cannot open another criterion

This is intentional. Only Criterion 5 is implemented. Enable another option only after its real API and renderer exist.


## Guided Student Journey question fails with `OPENAI_API_KEY is not defined`

Version 1.1.0 initializes the optional key to an empty value before routing.
Guided questions then use deterministic intent detection and ERPNext extraction without OpenAI.
Only optional free-form AI interpretation should provide an `openai_api_key`.

## Browser-console messages seen during v1.8.1 testing

The package source was searched and contains no references to
`survey.min.css`, `custom_calendar_override9.js` or `228.js`.

- `survey.min.css` 404: missing asset from another installed app or site build.
- `custom_calendar_override9.js` null `doctype`: separate Custom Script or app
  code running without the expected form context.
- repeated objects from `228.js`: separate compiled asset logging to the console.
- `runtime.lastError` message-channel errors: commonly generated by a browser
  extension or an unrelated asynchronous listener.
- unused preload warnings: browser performance warnings, not a Custom HTML Block
  runtime failure.

These issues should be investigated separately from the UCC Intelligence
Platform block.
